xv6 Operating System Enhancements

This project extends the xv6 Operating System, a simple Unix-like OS developed for educational purposes. The enhancements include the implementation of advanced scheduling algorithms, paging, and file system features. The project was initially developed as part of an operating systems lab, showcasing practical skills in OS development.

Enhancements and Features

1. Lottery Ticket Scheduling

  • Implemented lottery scheduling, a probabilistic CPU scheduling algorithm.
  • Processes receive “lottery tickets” proportional to their priority; the scheduler randomly selects a ticket to decide which process runs.
  • This approach ensures fair CPU allocation while allowing dynamic adjustment of process priorities.

2. Multi-Level Feedback Queue (MLFQ) Scheduling

  • Added MLFQ scheduling, a priority-based scheduler with multiple queues.
  • Processes dynamically move between queues based on their execution behavior (e.g., CPU-bound or I/O-bound tasks).
  • Balances between responsiveness and throughput, making it suitable for mixed workloads.

3. Paging

  • Implemented paging to support virtual memory management.
  • Used page tables to map virtual addresses to physical memory, enabling efficient memory utilization and protection.
  • The implementation allows the OS to handle out-of-memory scenarios more effectively, supporting larger workloads.

4. File System Enhancements

  • Enhanced the xv6 file system to improve functionality and performance.
  • Added support for advanced features like file metadata handling and optimized disk I/O operations.
  • This implementation bridges the gap between the simplistic xv6 file system and real-world OS requirements.

About the Project

This project demonstrates a deep understanding of operating system design and implementation.
Originally developed as part of an Operating Systems Lab, it highlights proficiency in scheduling algorithms, memory management, and file systems.
