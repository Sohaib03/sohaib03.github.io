Yet Another C Compiler

This repository contains my implementation of a C Compiler, developed as part of a lab project during my undergraduate studies at BUET. The project encompasses the complete pipeline of compiler construction, progressing through various stages to produce a functional compiler.

Compiler Stages

1. Symbol Table Creation

  • Designed and implemented a hashtable-based symbol table in C++ for efficient management of identifiers.
  • The symbol table supports storing and retrieving variable names, function signatures, and associated attributes.
  • This module forms the foundation for subsequent compiler phases, enabling efficient semantic analysis.

2. Lexical Analysis

  • Built a lexical analyzer to tokenize the input source code into a sequence of meaningful tokens.
  • Used Flex, a fast and powerful lexical analyzer generator, to define tokenization rules.
  • Flex converts source code into tokens like identifiers, keywords, and operators, making the code parsable by the next stage.
  • Flex was chosen over Lex due to its improved performance and better compatibility with modern C++ compilers.

3. Parsing

  • Constructed a parser to perform syntactic analysis on the tokenized input, ensuring it adheres to the language grammar.
  • Used Bison, a parser generator, to define a formal context-free grammar for C.
  • Bison constructs a parse tree and identifies syntax errors in the input code during this phase.
  • Parsing is designed as a modular stage, allowing easy updates to support more complex C language features.

4. Intermediate Code Generation (ICG)

  • Developed the Intermediate Code Generator to produce code compatible with the EMU8086 emulator.
  • This stage translates high-level C constructs into intermediate representations like three-address code (TAC).
  • Optimized the intermediate code to reduce redundancy and improve performance during execution.
  • The intermediate code serves as a bridge between source code and low-level assembly language.

About the Project

This C Compiler demonstrates the complete process of compiler construction, from symbol table management to code generation.
While originally created as part of the CSE 310 Compiler Lab at BUET, the project is a standalone tool, showcasing proficiency in compiler design and implementation.
